# 本机写作工坊 · UX 设计方案

> 日期：2026-05-20
> 目标：一个跑在本机的 Python Web 应用，整合 AI 聊天 + 写作工具箱 + 自动审查

## 一、项目目录结构

```
writing-app/
│
├── app.py                      # 入口，Flask 应用初始化 + 启动
├── config.py                   # 配置加载（.env -> 全局常量）
├── requirements.txt            # pip 依赖锁定
├── .env.example                # 环境变量模板（提交用）
├── .env                        # 实际环境变量（不提交）
├── README.md                   # ✅ 使用说明 = 用户第一份文档
├── run.bat                     # Windows 一键启动脚本
│
├── core/                       # 业务逻辑层
│   ├── __init__.py
│   ├── ai_chat.py              # AI 聊天引擎（OpenAI 兼容接口）
│   ├── writing_tools.py        # 写作工具箱（字数/可读性/风格统计）
│   ├── review_engine.py        # 自动审查引擎（拼写/语法/一致性）
│   └── doc_processor.py        # 文档读写（md/txt/docx 导入导出）
│
├── routes/                     # Flask 路由层
│   ├── __init__.py
│   ├── chat_routes.py          # /api/chat/*    聊天接口
│   ├── editor_routes.py        # /api/doc/*     文档增删改查
│   ├── review_routes.py        # /api/review/*  审查触发/结果
│   └── settings_routes.py      # /api/settings/* 配置读写
│
├── models/                     # 数据模型（SQLite）
│   ├── __init__.py
│   ├── project.py              # 项目/文档模型
│   └── session.py              # 聊天/审查记录
│
├── static/                     # 前端资源
│   ├── css/
│   │   ├── main.css            # 全局样式 + 布局
│   │   ├── editor.css          # 编辑器专属样式
│   │   └── chat.css            # 聊天面板样式
│   ├── js/
│   │   ├── app.js              # 主应用状态管理 + 路由
│   │   ├── editor.js           # 编辑器交互（CodeMirror/Markdown）
│   │   ├── chat.js             # 聊天交互（消息流+流式输出）
│   │   └── review.js           # 审查面板（结果展示+定位跳转）
│   └── lib/                    # 前端库（本地保留，避免 CDN 依赖）
│       └── marked.min.js
│
├── templates/                  # Jinja2 模板
│   ├── base.html               # 布局骨架（nav + 主区域 + 侧栏）
│   ├── index.html              # 主工作台页面
│   ├── settings.html           # 设置页
│   └── welcome.html            # ✅ 首次引导页
│
├── chapters/                   # 写作内容（md 文件，按项目分目录）
├── knowledge/                  # 知识库文件（参考文档/大纲/设定集）
└── data/                       # 运行时数据
    ├── writing.db              # SQLite 数据库（自动创建）
    └── .gitkeep
```

### 目录设计原则

| 层级 | 职责 | 说明 |
|------|------|------|
| `app.py` | 入口 + 启动 | 一个文件，不做拆分 |
| `core/` | 纯业务逻辑 | 不感知 Flask，可单独单元测试 |
| `routes/` | HTTP 接口适配 | 薄层，只做参数解析 + 调用 core + 返回 |
| `models/` | 数据持久化 | ORM 模型，与 routes 解耦 |
| `static/` + `templates/` | 前端 | SPA 风格，后端只做初始渲染和 API |

## 二、用户从下载到启动的步骤

### Step 1：确认环境
```
python --version      # 需 >= 3.10
git --version         # 可选，用于克隆
```

### Step 2：获取项目
```
git clone <仓库地址>
cd writing-app
```

### Step 3：创建虚拟环境
```
python -m venv venv
```
Windows 激活：`venv\Scripts\activate`
非 Windows：`source venv/bin/activate`

### Step 4：安装依赖
```
pip install -r requirements.txt
```

### Step 5：配置
```
copy .env.example .env      # Windows
# 或
cp .env.example .env        # POSIX
```
然后编辑 `.env`，填入 AI API 的 base_url 和 api_key。
> 不配置也能运行——编辑器、审查、本地工具全可用，仅 AI 聊天不可用。

### Step 6：启动
```
# 方式一：直接运行
python app.py

# 方式二：双击脚本
run.bat
```

### Step 7：打开浏览器
终端输出：
```
 * Running on http://127.0.0.1:5000 (Press CTRL+C to quit)
```
点击或复制地址到浏览器。

### 一键脚本（run.bat）
```bat
@echo off
cd /d "%~dp0"
if not exist venv\Scripts\activate (
    echo 首次运行，正在安装依赖...
    python -m venv venv
    call venv\Scripts\activate
    pip install -r requirements.txt
) else (
    call venv\Scripts\activate
)
python app.py
pause
```

## 三、界面功能层级

### 一级入口（主工作台，始终可见）

| 区域 | 功能 | 交互说明 |
|------|------|----------|
| **标题栏** | 项目名 + 保存状态 | 显示当前项目，未保存时显示红点 |
| **主编辑区** | Markdown 编辑器 | 占据 60% 以上宽度，双栏模式（编辑+预览） |
| **AI 聊天** | 侧边浮动面板 | 可收起/展开，支持流式输出，上下文关联当前文档 |
| **审查面板** | 底部/右侧面板 | 列出问题列表，点击定位到原文 |

这四项是用户**每次打开都在用的**，放在一级，零跳转到达。

### 二级入口（通过菜单/按钮/设置页面进入）

| 功能 | 上级入口 | 触发方式 |
|------|----------|----------|
| 新建项目 | 标题栏左上角"写作工坊"下拉菜单 | 点击 |
| 打开已有项目 | 同上 | 点击 |
| 保存 / 另存为 | 标题栏保存按钮 | 点击 |
| 导出（md/pdf/docx） | 标题栏"···"更多菜单 | 点击 |
| 历史版本 | 编辑器右上角时钟图标 | 点击 |
| 写作统计 | 底部状态栏统计图标 | 点击 / 悬停 |
| 知识库管理 | 标题栏"···"更多菜单 | 点击 |
| 设置 | 用户头像/齿轮图标 | 点击 |
| 审查规则配置 | 审查面板右上角齿轮 | 点击 |

## 五、补充设计备注

### 离线降级
- AI 聊天功能依赖 `.env` 配置，未配置时聊天界面显示"API 未配置"提示 + 设置跳转链接
- 编辑器、审查（基础版）、写作工具、导入导出完全离线可用

### 审查引擎两层设计
- **快速审查（本地）**：正则+规则引擎，毫秒级反馈，覆盖拼写、标点、长句
- **深度审查（AI）**：调用 API 分析连贯性、风格一致性、节奏，有配置才可用

### 快捷键设计
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+S` | 保存 |
| `Ctrl+Shift+L` | 选中文本送入 AI 聊天 |
| `Ctrl+Shift+R` | 触发审查 |
| `Ctrl+F` | 编辑器内搜索 |
| `Ctrl+B` | 加粗（编辑器内） |

### 数据存储策略
- SQLite 存项目元数据、聊天记录、审查历史
- 正文存 `.md` 文件在 `chapters/` 下，按 `{项目名}/{章节名}.md` 组织
- 这样做的好处：即使用 SQLite 坏了，文稿文件还在，可以直接用其他编辑器打开
