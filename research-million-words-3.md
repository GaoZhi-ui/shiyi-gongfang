## 一、完本案例：明日方舟同人长篇

### 已完结 / 百万字级

| 作品 | 平台 | 字数 | 状态 | 特点 |
|------|------|------|------|------|
| **《明天之梦》** | SF轻小说 | 268万字 | 已完结 | 舟同人，因部分原因停更后改名《明日方舟之归途》重启 |
| **《明日方舟之深海的呼唤》** | QQ阅读 | 117万字 | 完结+番外 | 阿戈尔主线，歌蕾蒂娅相关 |
| **《人在方舟，屑到失踪》** | 刺猬猫 | 428.6万字 | 已完结 | 428万字级超长篇完本，商业同人 |
| **《明日方舟：支点之末》** | 世纪文库 | 255.86万字 | 连载中 | 前文明存续计划，签约作品 |
| **《明日方舟的革命者》** | 刺猬猫 | 32.7万字 | 已完结 | 地球穿越者，切尔诺伯格革命线，较短但结构完整 |
| **《[综+明日方舟]模板是夜莺》** | 晋江 | 33.8万字 | 完结 | 综漫型，非单一方舟 |
| **《明日方舟，萃香的信使之役》** | 同人小说网 | 100+万字 | 连载中 | 幻想乡x方舟 crossover |

### 关键发现

1. **方舟同人百万字完本确实存在**，但主要集中在刺猬猫、SF轻小说等二次元平台。
2. 428万字的《人在方舟，屑到失踪》是目前发现的方舟同人最长完本——证明同人写到400万字以上是可行的。
3. 大部分百萬字级作品处于"连载中"状态，真正的完本不多。刺猬猫以"鸽子窝"著称，能完本就是质量的证明。
4. 32~34万字是许多同人作品的"自然完结长度"——要从这个长度突破到百万字，需要故事规模本身的扩展。

### 2.3 已验证的百万字写作流程

#### 分层式大纲体系（马良写作 + 多位作者实战验证）

```
第一层：总纲（稳定不変）
   ├── 核心矛盾
   ├── 主角成长弧线
   └── 大结局方向

第二层：卷纲（每卷开始时重写）
   ├── 基于前卷实际输出内容重新规划
   └── 而非最初设想

第三层：章节蓝图（每章调整）
   ├── 写完上一章後微调下一章
   └── 与知识图谱联动
```

#### Claude Code 70万字小说生产流程（2025年案例）

1. **市场分析** → 去番茄读书观察热门榜单，分析题材趋势
2. **竞品学习** → 投喂同题材小说，分析开头套路、人设、冲突模式
3. **苏格拉底式讨论** → 和AI反复讨论题目和大纲
4. **Wiki/RAG/记忆文件** → 为长篇建立设定库
5. **单章生产循环**：
   - 章节计划（目标、钩子、冲突、伏笔）
   - 检查计划（是否水、有无信息增量）
   - 写正文初稿（2500-3500字）
   - 审稿（6项自检：水字数/主角主动/怪物换皮/AI总结/追读钩子/主线推进）
   - 更新Wiki、记录新伏笔

#### 日更6000字的AI辅助流水线（WorkBuddy案例）

1. AI润色小说大纲
2. 从细纲批量生成章节卡
3. AI辅助正文写作 + 去AI味
4. 章节一致性检查
5. 违禁词和敏感内容扫描

#### 核心原则：人主导，AI辅助

- 人明确核心要素（题材、爽点、人物）
- AI负责细化填充
- 人写开篇1000字确定钩子，AI优化节奏
- 人想清楚章节核心事件，AI写出初稿
- AI生成内容必须人工修改至少30%

## 三、百万字级别角色弧光设计

### 3.1 三种核心弧光类型

| 类型 | 路径 | 适用 | 经典参考 |
|------|------|------|----------|
| **成长型** | 缺陷→经历→成熟 | 励志成长题材 | 《哈利·波特》《火影忍者》 |
| **堕落型** | 善良→受挫→黑化 | 悲剧美学题材 | 《绝命毒师》《进击的巨人》艾伦 |
| **平面型** | 核心价值观始终如一，改变世界 | 英雄史诗题材 | 《一拳超人》埼玉、《肖申克的救赎》安迪 |

### 3.2 五大构建要素

1. **初始状态**：定义核心缺陷与内在谎言
2. **催化剂**：冲突迫使面对内在矛盾
3. **转变节点**：25%面对缺陷 → 50%认知质变 → 75%关键选择
4. **情感动因**：选择基于情感需求
5. **结局状态**：改变付出代价，形成逻辑闭环

### 3.3 实战操盘模板

```
角色弧光设计表
──────────────────────────────────────
初始状态：______（缺陷 + 谎言）
催化剂：  ______（触发事件）
转变节点：
  25% ______
  50% ______
  75% ______
结局状态：______（新平衡 + 代价）
──────────────────────────────────────
```

### 3.4 百万字级别的特殊注意事项

**角色状态时间线**
- 每次角色出场后，维护其状态变化
- 谁受伤了、谁突破了、谁和谁结盟了
- 写到50万字以上时，定期做"设定审计"

**伏笔管理**
- 每埋一条伏笔，立刻在大纲标注"预计在第X章回收"
- AI辅助工具（马良写作等）支持伏笔关联功能

**支线控制**
- 同时展开的支线不超过3条
- 完成一条再开新的

**每卷钩子**
- 这一卷的大Boss打完了，战斗中暴露一个更大的秘密
- 这个秘密就是下一卷的引子

### 3.5 避免"人设崩塌"的方法

- 控制一章同时出场角色不超过4-5个
- 每个角色给"语言指纹"（口头禅、说话方式）
- AI生成时自动注入角色卡作为上下文基准
- 用行为代替心理描写（"他把烟点了三次都没点着"）

## 四、百万字时间规划

### 4.1 速度基准

| 水平 | 时速 | 日更（2-3小时） | 日更（4-6小时） |
|------|------|----------------|----------------|
| 纯新人 | 500-1000字 | 1000-3000字 | 2000-6000字 |
| 有经验 | 2000-3000字 | 4000-9000字 | 8000-18000字 |
| 专业作者 | 3000-4000字 | 6000-12000字 | 12000-24000字 |
| 头部作者 | 4000+字 | 8000+字 | 20000+字 |

> 行业标准：日更4000字是入门标配，头部作者日更万字是常态。
> AI辅助作者平均日更8900字，纯人工平均日更3200字。

### 4.2 百万字所需时间推算

| 日更量 | 达成100万字 |
|--------|------------|
| 2000字 | 500天（1.4年） |
| 3000字 | 334天（11个月） |
| **4000字** | **250天（8个月）** ← 行业标配 |
| 6000字 | 167天（5.5个月） |
| 8000字 | 125天（4个月） |
| 10000字 | 100天（3.3个月） |

### 4.3 真实作者案例

| 作者/案例 | 作品字数 | 耗时 | 日更量 | 备注 |
|-----------|---------|------|--------|------|
| 墨香铜臭《魔道祖师》 | ~70万字 | 大纲半年+写作5月≈1年 | ~4000字 | 80%按大纲，20%临时灵感 |
| 网文作者（知乎） | 150万字 | 1年多 | ~4000字 | 第一本百万字，总稿费4万+ |
| Claude Code 作者 | 70万字 | 1个假期（约3个月） | ~8000字 | AI全流程，含市场分析 |
| Reddit 作者 | 10万字 | 1个月 | ~3300字 | 大纲1周+写作3周 |
| 番茄写手（记录帖） | 100万字目标 | 计划6个月 | 6000字 | 偏日常向，节奏慢 |
| 晋江作者（记录帖） | 100万字目标 | 规划中 | 日更3000 | 每日3小时输出 |

### 4.4 同人写作的特殊速度影响

- 同人不需原创世界观，设定成本低于原创网文
- 但同人需要更严格的人设一致性，避免OOC
- 方舟同人有官方剧情参考，素材丰富
- 《泰拉拾遗录》属于"地球人走入泰拉"型，介于同人和原创之间

### 4.5 关键经验总结

**来自多位百万字作者**：

1. **写过10万字和写过100万字，状态完全不同**。写到七八十万字时才能真正理解自己缺什么。
2. **长篇最大的敌人不是卡文，是心理落差**——成绩、读者反馈、写作难度的变化会产生巨大压力。
3. **大纲的构思时间比写作时间长很正常**。墨香铜臭大纲构思半年，写作五个月。
4. **每一卷开始时重写卷纲**，基于实际写出来的内容重新规划，而不是死守最初设想。
5. **每10万字做一次设定审计**，能抓出3-5个小矛盾。提前修正比读者找出来强一万倍。
6. **日更3000-4000字是可持续的节奏**。高于6000字需要全职或AI辅助。
7. **写不完的最大原因：故事规模太小**。100万字的自然长度需要足够大的世界和足够多的角色。
