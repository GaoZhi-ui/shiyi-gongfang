# 泰拉拾遗录 · 写作工坊

《泰拉拾遗录》专属写作环境。一个跑在本机的 Python Web 应用，整合 AI 聊天 + 写作工具箱 + 自动审查。

---

## 项目简介

写作工坊是围绕《泰拉拾遗录》创作流程搭建的辅助工具。三个核心能力：

- **写** — Markdown 编辑器，左侧编辑右侧实时预览
- **聊** — AI 助手，随时问答、续写、诊断节奏
- **审** — 自动审查，逐项检查拼写、句式、字数、密度

编辑器、审查（基础版）、写作工具完全离线可用。AI 聊天需配置 API Key。

## 安装步骤

### 环境要求

- Python 3.10+
- 操作系统：Windows / macOS / Linux

### 方式一：一键启动（推荐）

双击 `run.bat`，脚本会自动完成以下工作：

```
1. 检测 Python 版本         → 需 ≥ 3.10
2. 创建虚拟环境 venv         → 仅首次
3. 安装 pip 依赖             → 仅首次
4. 启动 uvicorn 服务         → http://localhost:8000
5. 自动打开浏览器             → 完成
```

### 方式二：手动启动

```bash
# 1. 创建虚拟环境
python -m venv venv

# 2. Windows 激活
venv\Scripts\activate
# macOS/Linux 激活
# source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动服务
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

浏览器访问 `http://localhost:8000`。

### 方式三：二进制分发（PyInstaller 打包）

无需 Python 环境，下载对应平台的压缩包解压即用。

| 平台 | 下载 |
|------|------|
| Windows 10+ | `拾遗工坊.exe` (dist/ 目录) |
| macOS 12+ | `拾遗工坊` (dist/ 目录) |
| Linux (x86_64) | `拾遗工坊` (dist/ 目录) |

```bash
# 自行构建二进制
# Windows
python build.py --clean

# Linux / macOS
chmod +x build.sh && ./build.sh --clean
```

## 首次使用引导

### 第 1 步：欢迎页

打开应用后看到欢迎页，介绍写作工坊的三个核心能力。点击"开始设置"进入配置，或"跳过，直接使用"进入主工作台。

### 第 2 步：配置 AI

在设置页面填入 API Key 和接口地址（默认 DeepSeek）。

```
API 接口地址：https://api.deepseek.com/v1
API Key：     从 DeepSeek 官网获取
模型：        deepseek-chat
```

不配置也能使用编辑器、审查、写作工具。仅 AI 聊天不可用，随时可以在设置中补上。

### 第 3 步：创建第一个项目

输入项目名称，选择一个起始模板：

| 模板 | 适用场景 |
|------|----------|
| 空白文档 | 自由创作 |
| 小说章节 | 包含标题 + 元数据头 |
| 随笔/日记 | 轻松写作 |
| 知乎回答草稿 | 问答体写作 |

### 第 4 步：快速导览

进入主工作台后，会依次高亮提示：

1. **编辑器区域** — 左侧写 Markdown，右侧实时预览
2. **AI 聊天侧栏** — 需要灵感时打开，选中文字按 `Ctrl+Shift+L` 送入聊天
3. **审查面板** — 写完点击审查，检查拼写、句式和连贯性

引导只出现一次。再次打开直接进入上次编辑的项目。

## 功能说明

### 主工作台布局

```
┌─────────────────────────────────────────────────────┐
│  [≡] 写作工坊  │  泰拉拾遗录·第31章  │  💾  […]   │  ← 标题栏
├──────────────────────────┬──────────────────────────┤
│                          │                          │
│    Markdown 编辑器       │   实时预览               │  ← 主编辑区
│                          │                          │
│                          │                          │
├──────────────────────────┴──────────────────────────┤
│  ⚠ 审查发现 3 个问题  [拼写×1] [句式×2]  [定位]   │  ← 审查面板（可折叠）
├─────────────────────────────────────────────────────┤
│  字数: 2,340  │  阅读时间: ~9min  │  词频统计  ▸   │  ← 状态栏
└─────────────────────────────────────────────────────┘
                                          ┌─────────┐
                                          │  AI 聊天 │  ← 浮动侧栏（可收起）
                                          └─────────┘
```

### 编辑器

- 左侧编辑 Markdown，右侧实时预览
- 支持双栏/单栏切换
- 快捷键：`Ctrl+S` 保存，`Ctrl+F` 搜索，`Ctrl+B` 加粗
- 选中文本按 `Ctrl+Shift+L` 送入 AI 聊天

### AI 聊天

- 侧边浮动面板，可收起/展开
- 流式输出，逐字显示回复
- 上下文关联当前文档
- 支持多轮对话

支持以下 Provider：

| Provider | 模型 | 接口地址 |
|----------|------|----------|
| DeepSeek | deepseek-chat | https://api.deepseek.com/v1 |
| OpenAI | gpt-4o, gpt-4o-mini | https://api.openai.com/v1 |
| Anthropic | claude-sonnet-4 | https://api.anthropic.com |

### 自动审查

- **快速审查（本地）** — 正则 + 规则引擎，毫秒级反馈
- **深度审查（AI）** — 调用 API 分析连贯性、风格一致性（需配置 Key）

审查项目：字数、句号密度、句式重复（"不是…是…"）、"他"字频次、"好的"过渡、日记完整性、末句钩子

### 知识库

侧栏预设的《泰拉拾遗录》知识库入口：

| 分组 | 内容 |
|------|------|
| 写作流程 | 9阶段流程、自检清单、自动审查 |
| 知识库 | 工具箱一览、写作风格速查、人物档案 |
| 章节管理 | 全卷细纲、时间线表、伏笔追踪 |

### 状态栏

底部状态栏显示字数、阅读时间、词频统计，点击可展开详细分析。

### 设置

- API Key 配置（加密存储，绑定本机）
- 模型选择
- 主题切换（暗色/亮色）

## 项目架构

```
┌──────────────────────────────────────────────────┐
│                 用户层 (浏览器)                    │
│   HTML/CSS/JS  (Markdown 编辑器 + AI 聊天界面)    │
├──────────────────────────────────────────────────┤
│                 API 层 (FastAPI)                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 健康检查  │ │ 密钥管理  │ │ 知识库 / 章节管理 │  │
│  ├──────────┤ ├──────────┤ ├──────────────────┤  │
│  │ AI 聊天  │ │ 写作工具  │ │ 工作流引擎       │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
├──────────────────────────────────────────────────┤
│               业务层 (Services)                    │
│  审查引擎 | 加密存储 | 模板渲染 | 词频统计        │
├──────────────────────────────────────────────────┤
│                数据层                              │
│  SQLite (运行时) | 文件系统 (.md 章节) | 本地缓存  │
└──────────────────────────────────────────────────┘
```

| 层级 | 职责 | 技术选型 |
|------|------|----------|
| **用户层** | Markdown 编辑器、实时预览、AI 聊天 UI | 原生 HTML/CSS/JS + Marked.js |
| **API 层** | RESTful 接口，路由分发 | FastAPI + uvicorn |
| **业务层** | 核心逻辑：审查、加密、模板、工作流 | Python（无外部框架依赖） |
| **数据层** | 持久化存储与运行时状态 | SQLite (aiosqlite) + 文件系统 |

### API 路由结构

| 前缀 | 路由模块 | 功能 |
|------|----------|------|
| `/api/v1/health` | `routers/health.py` | 服务健康检查 |
| `/api/v1/keys` | `routers/keys.py` | API Key 加密管理 |
| `/api/v1/knowledge` | `routers/knowledge.py` | 知识库查询 |
| `/api/v1/chapters` | `routers/chapters.py` | 章节 CRUD |
| `/api/v1/chat` | `routers/chat.py` | AI 流式聊天 |
| `/api/v1/tools` | `routers/tools.py` | 写作工具箱 |
| `/api/v1/workflow` | `routers/workflow.py` | 写作工作流引擎 |

### 数据流示例：写入 + 审查

```
用户在编辑器输入 Markdown
       ↓
Ctrl+S 触发保存 → POST /api/v1/chapters → 写入 SQLite + .md 文件
       ↓
Ctrl+Shift+R 触发审查 → POST /api/v1/tools/review
       ├─ 快速审查 (本地正则引擎, 毫秒级) → 返回拼写/句式/字数问题
       └─ 深度审查 (如配置了 API Key) → 调用 AI 分析连贯性/风格
       ↓
审查结果展示在底部面板
```

## 贡献指南

欢迎贡献！无论是 Bug 修复、功能建议还是文档改进，请遵循以下流程。

### 开发环境

```bash
# Fork 后克隆
# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装开发依赖
pip install -r requirements.txt
pip install pyinstaller  # 构建用

# 启动开发服务器
uvicorn main:app --reload --port 8000
```

### 分支规范

| 分支 | 用途 |
|------|------|
| `main` | 稳定版，保持可发布状态 |
| `dev` | 日常开发，功能集成 |
| `feature/*` | 新功能分支，从 dev 切出 |
| `fix/*` | Bug 修复分支 |

### 提 PR 之前

- [ ] 代码通过自审（`python -m py_compile main.py`）
- [ ] 新增功能包含对应的测试或审查规则
- [ ] README 已同步（如有界面或 API 变化）
- [ ] 遵循现有代码风格：4 空格缩进、清晰命名、适量注释

### 功能开发流程

1. 从 `dev` 切出 `feature/你的功能名`
2. 实现功能，保持每个 commit 原子化
3. 更新 README 和相应的 API 设计文档
4. 提交 PR 到 `dev` 分支

### 反馈与请求

提 Issue 时请注明：

- 操作系统与 Python 版本
- 复现步骤（如适用）
- 期望行为 vs 实际行为

## 常见问题

**Q：启动后浏览器打不开？**
A：手动访问 http://localhost:8000。

**Q：AI 聊天显示"API 未配置"？**
A：去设置页面填入 API Key。没有 Key 也能用编辑器和审查。

**Q：端口 8000 被占用了？**
A：编辑 run.bat 最后一行，把 `--port 8000` 改成其他端口。

**Q：需要联网吗？**
A：编辑器、审查、写作工具完全离线。仅 AI 聊天需要联网调用 API。
