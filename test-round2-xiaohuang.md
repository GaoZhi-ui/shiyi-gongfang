## 额外发现

### 🔴 问题1: 项目删除权限错误

清理测试数据时发现，所有项目 DELETE 请求均返回 **500**：

```
DELETE /api/v1/projects/{id} → 500
"code":"PROJECT_DELETE_FAILED","message":"删除项目失败: [WinError 5] 拒绝访问"
```

根因是项目目录被服务器进程锁定或权限不足。这影响用户正常管理项目的功能。

### 🔴 问题2: 不存在的项目导出静默返回数据

1c 测试中，传入完全不存在的项目名和章节名，服务端仍然返回了201和一个有效文件。这表明导出功能可能没有验证输入参数与存储数据的关联性，存在"凭空生成"的可能。

### 🟡 问题3: XSS 项目名未过滤

`<script>alert(1)</script>` 作为项目名被直接存入。虽然 API 返回 JSON 不会直接执行，但若：
- 前端渲染项目列表时未做转义
- 项目名出现在文件路径中
- 导出文档的元数据中包含项目名

则存在 XSS 或注入风险。

### 🟡 问题4: 字段长度校验不一致

| 字段 | 限制 | 状态 |
|------|------|------|
| project.name | max 64 | ✅ 有校验 |
| project.description | 无限制 | ⚠️ 无校验 |
| chapter.content | 无限制 | ⚠️ 无校验 |

建议对 description 和 content 设置合理上限（如 64KB / 1MB）。
