## 三、自部署AI界面类

### Open WebUI

**定位**：通用自部署AI聊天前端，面向所有AI交互场景，非写作专用。

**我们有而它没有的**：
- 专门的写作流程（9阶段写作管线）
- 自检清单和知识库文件体系
- 自动审查（字数/密度/句式等中文写作特有检查）

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| 本地RAG（9种向量数据库） | 支持文档直接载入对话，用 `#` 命令引用 | ⭐⭐⭐ 高 — 替代/增强现有的知识库文件检索 |
| Web搜索集成（15+供应商） | 在对话中直接搜索并注入结果 | ⭐⭐ 中 — 写作调研场景有用 |
| Pipelines插件框架 | 自定义Python逻辑注入，Function Calling | ⭐⭐ 中 — 可扩展审查规则 |
| 图像生成集成 | DALL·E / ComfyUI / SD 直接出图 | ⭐ 低 — 写作辅助非必需 |
| 语音/视频通话 | 多STT/TTS引擎 | ⭐ 低 — 写作场景非必需 |
| RBAC多用户/企业认证 | LDAP/SSO/SCIM | ❌ 不适用 — 单用户工具 |

### LobeHub（含Lobe Chat演进）

**定位**：AI Agent协作平台，以Agent为单位组织工作，强调人机协同进化。

**我们有而它没有的**：
- 写作工作流（9阶段管线）
- 专用写作审查工具
- 章节管理

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| Pages（多Agent协作写作） | 多个Agent在同一上下文协同撰写和修订内容 | ⭐⭐⭐ 高 — 可借鉴"多角色AI写作助手"概念 |
| Agent Builder / 自定义角色 | 描述需求自动生成专属写作Agent | ⭐⭐ 中 |
| Personal Memory / 持续学习 | Agent学习用户写作习惯，白盒可编辑记忆 | ⭐⭐ 中 — 可用于风格记忆 |
| 分支对话 | 对话树分支，探索不同剧情方向 | ⭐⭐ 中 — 写作分支剧情时有用 |
| 10,000+插件 / MCP | 大规模插件市场 | ⭐ 低 — 但架构思路可参考 |
| Artifacts支持 | AI生成的富内容（代码/文档）独立渲染 | ⭐ 低 |

## 四、传统写作软件

### Scrivener

**定位**：长篇写作项目管理工具，被视为"作家的IDE"。

**我们有而它没有的**：
- AI对话集成（它完全无AI能力）
- 自动化审查检查
- 预置写作流程和自检清单

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| **Corkboard + 索引卡片** | 可视化故事板，卡片可拖拽排序，每张卡片含概要/状态/标签 | ⭐⭐⭐ 高 — 视觉化的章节概览是写作软件的标志性功能 |
| **Snapshots（快照版本）** | 修改前拍快照，随时回退到任意历史版本 | ⭐⭐⭐ 高 — 写作中频繁改稿，回退能力很重要 |
| **关键词标签+筛选** | 给文档打多个标签（角色/地点/时间线等），按标签筛选排序 | ⭐⭐⭐ 高 — 对复杂长篇非常有用 |
| **Compile编译引擎** | 一键编译为ePub/Mobi/PDF/DOCX/HTML，支持自定义格式模板 | ⭐⭐ 中 — 目前MD→docx手动转换可改进 |
| **模板系统** | 小说/剧本/非虚构/学术等预设模板 | ⭐⭐ 中 |
| **分屏编辑** | 同时查看/编辑多个文档 | ⭐ 低 |
| **目标跟踪** | 项目总字数目标，章节独立字数目标 | ⭐⭐ 中 — 拾遗工坊已有审查字数，可扩展为目标追踪 |

### Ulysses

**定位**：macOS/iOS生态的沉浸式Markdown写作环境。

**我们有而它没有的**：
- AI对话和AI辅助写作
- 中文写作审查（句式/密度等）
- 多API供应商支持

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| **智能过滤组** | 基于关键字/日期/状态自动过滤文稿，类似智能文件夹 | ⭐⭐ 中 — 按"待修改/已完成"自动分类 |
| **修订模式** | 样式检查（长句提醒、重复开头词、替代词建议） | ⭐⭐ 中 — 与现有审查功能互补 |
| **写作目标** | 每日字数目标 + 单篇进度环 | ⭐⭐ 中 — 配合写作管理 |
| **直接发布** | 一键发布到WordPress/Ghost/Medium | ⭐ 低 — 写作阶段不急需 |
| **打字机模式** | 正在编辑的行始终居中，保持焦点 | ⭐ 低 |
| **第二编辑器** | 对照翻译/对比编辑 | ⭐ 低 |

### yWriter

**定位**：免费的长篇小说写作软件，作者出身程序员开发。

**我们有而它没有的**：
- AI能力（它完全无AI）
- 自动审查
- 知识库管理

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| **场景级管理** | 场景是基本单元（比章节更细粒度），每个场景可绑定额外的角色/地点/情感弧线 | ⭐⭐⭐ 高 — 拾遗工坊现有章节管理，增加场景层级更精细 |
| **角色/地点数据库** | 独立DB管理角色、地点、物品，与场景双向关联 | ⭐⭐ 中 — 知识库已涵盖，但关联展示可以更直观 |
| **故事板浏览** | 可视化所有场景的时间线分布 | ⭐⭐ 中 |
| **项目统计** | 各章节字数/场景数/角色出场统计 | ⭐ 低 — 现有审查已有部分 |

## 五、AI写作工具

### Sudowrite

**定位**：面向小说作者的AI写作搭档，强调"在你风格中写作"。

**我们有而它没有的**：
- 本地运行，API Key加密（它是纯SaaS）
- 中文写作审查（Sudowrite对中文支持弱）
- 多AI供应商（它仅绑OpenAI）
- 知识库文件体系

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| **Story Bible** | 从创意→大纲→章节细纲的逐步引导流程，AI驱动 | ⭐⭐⭐ 高 — 与9阶段流程互补，可用AI辅助填充细纲 |
| **Write（风格感知续写）** | 分析角色/调性/情节弧线后，以你的风格续写300词 | ⭐⭐⭐ 高 — 比通用对话更专注写作场景 |
| **Describe / Expand** | 感官描写增强/场景扩展专用工具 | ⭐⭐⭐ 高 — 对标"文笔润色"功能可更精细化 |
| **Rewrite（多版本重写）** | 一次生成多个重写版本供选择 | ⭐⭐⭐ 高 — 改稿利器 |
| **Feedback（结构分析）** | 对选中内容给出3个可改进方向 | ⭐⭐ 中 — 与现有审查互补，但更AI驱动 |
| **Canvas（情节探索白板）** | 可视化情节地图，AI建议分支/转折/秘密 | ⭐⭐ 中 |
| **Visualize（角色可视化）** | 根据文字描述生成角色肖像 | ⭐ 低 |
| **Brainstorm** | 无限灵感激荡，可收藏偏好方向 | ⭐ 低 |
| **插件市场** | 1000+社区插件（模拟读者/角色对话/剧本转换等） | ⭐⭐ 中 — 架构思路，非功能本身 |

### Novelcrafter

**定位**：一站式AI小说创作与管理平台，重视故事构思的完整性。

**我们有而它没有的**：
- 本地运行，API Key加密
- 9阶段写作管线
- 自动审查（它是纯AI写作辅助，无审查功能）
- 中文写作支持

**它有而我们没有的**（可借鉴）：

| 功能 | 说明 | 建议优先级 |
|---|---|---|
| **Codex（设定自动关联）** | 角色/地点/传说/物品统一管理，AI自动识别并关联到正文 | ⭐⭐⭐ 高 — 突破现有知识库"手动引用"的限制 |
| **Scene Cards + 拖拽排序** | 场景卡片化，概览+拖拽调整顺序，支持多视角查看（Act/Chapter/Scene层级） | ⭐⭐⭐ 高 |
| **Chat → Extract** | AI对话内容一键提取为场景节拍/设定条目/章节 | ⭐⭐⭐ 高 — 减少从对话到正文的"搬运"成本 |
| **透明提示词系统** | 用户可查看和修改AI底层的提示词模板，自定义AI行为 | ⭐⭐ 中 — 高级用户需求 |
| **Review仪表盘** | 总字数/场景字数/角色出场时间线统计 | ⭐⭐ 中 |
| **Snippets（灵感片段）** | 随时记录灵感，灵活插入到场景 | ⭐⭐ 中 |
| **系列管理** | Codex跨书共享，系列作品统一设定 | ⭐ 低 — 目前单本为主 |
| **团队协作** | 邀请他人共同编辑/审阅 | ❌ 不适用 — 单用户工具 |

## 六、总结：拾遗工坊的定位与建议

### 当前差异化优势

1. **中文写作专属审查** — 密度检查、句式分析、日记/钩子检查，这些都是面向中文网络小说的特有功能，竞品无此能力
2. **9阶段写作管线** — 从写前分析到章末元数据的完整流程，竞品中仅Sudowrite的Story Bible和Novelcrafter的Plan有部分类似
3. **20个知识库文件体系** — 精细化的世界设定管理，比Novelcrafter的Codex更结构化，比Scrivener的研究文件夹更系统
4. **纯本地 + API Key加密** — 隐私和安全性优于所有SaaS工具

### 最具优先级的功能追加建议

按价值/实现成本排列：

| 优先级 | 功能 | 来源参考 | 预期价值 |
|---|---|---|---|
| **P0（必须补）** | 场景级管理（章节下分场景） | yWriter / Novelcrafter | 提高章节内部组织粒度 |
| **P0（必须补）** | 可视化场景看板/Corkboard | Scrivener / Novelcrafter | 直观掌握故事全局结构 |
| **P0（必须补）** | AI续写/扩写 / 重写 | Sudowrite Novelcrafter | 核心写作效率提升 |
| **P1（建议补）** | Codex式自动知识关联 | Novelcrafter | 减少手动关联成本 |
| **P1（建议补）** | Snapshots版本快照 | Scrivener | 改稿安全感 |
| **P1（建议补）** | Chat→写作内容一键提取 | Novelcrafter | 减少对话到正文的搬运 |
| **P1（建议补）** | 写作目标追踪 | Ulysses / Scrivener | 创作节奏管理 |
| **P2（可后续）** | 关键词标签+筛选 | Scrivener | 长篇信息检索 |
| **P2（可后续）** | 导出编译引擎 | Scrivener / Ulysses | 自动化交付流程 |
| **P2（可后续）** | 多Agent协作写作 | LobeHub | 多角色辅助写作 |
| **P3（低优先级）** | 插件市场/扩展框架 | Sudowrite / Open WebUI | 长期生态建设 |
| **P3（低优先级）** | 智能过滤组 | Ulysses | 自动分类 |

### 核心建议

拾遗工坊最独特的竞争力在于**针对中文长篇写作的深度流程支持**（9阶段管线+审查+知识库），这是通用AI前端和英文AI写作工具无法替代的。

追加功能时应优先强化**写作流程的闭环体验**：
- 场景级管理 + 可视化看板 → 让大纲从"树状列表"变成"可操作的视觉蓝图"
- AI续写/扩写/重写 → 让AI能力从"对话窗口"融入"写作管道"
- 版本快照 + 一键提取 → 减少编辑摩擦

**不建议**去追图像生成、语音通话、多用户协作等功能——这些偏离了核心写作场景，而且竞品也已经做得很好。拾遗工坊的战场在"中文创作流程的深度"，不在"功能数量的广度"。
