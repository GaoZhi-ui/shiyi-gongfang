## 二、尚未引入但有价值的前沿方向

### ① Local-First 同步架构（CRDT）

**来源**：HedgeDoc 2.0 迁移到 Yjs CRDT、Automerge、Local-First 运动

**核心思想**：用户的数据永远先存在本地，同步是后来的事。即使云端不可达，本地完全可用。

**对我们的价值**：
- 当前项目是单机本地存储，天然 Local-First
- 如果未来需要支持多设备，CRDT 是最干净的方案
- Yjs 是 JS 生态最成熟的 CRDT 库，可以直接前置到当前文件系统之上

**工作量**：4-6 周（如果要做多设备同步）
**引入时机**：当出现多设备需求时

---

### ② 向量化知识库（RAG 2.0）

**来源**：2025-2026 年 RAG 从简单检索进化为 Agentic RAG

**核心思想**：不仅仅是关键词搜索知识库，而是：
1. 章节内容自动向量化存入本地向量数据库
2. AI 在回答问题时自动检索相关章节
3. 跨章节的角色/地名一致性检查

**当前状态**：simple keyword search on .md files
**改进方向**：接入本地向量数据库（如 ChromaDB/lanceDB），章节保存时自动向量化

**工作量**：2-3 天
**价值**：AI 回答质量显著提升

---

### ③ 语义化 Git 集成（写作版本控制）

**来源**：NovelWriter 的纯文本 + @元数据标签天然 Git 友好

**核心思想**：每个项目文件夹就是一个 Git 仓库，每次保存自动 commit

**当前状态**：手动快照（snapshots API）
**改进方向**：
- 在项目创建时初始化 git repo
- 章节保存时 `git add . && git commit -m "auto: 第X章"`
- 快照对比直接用 `git diff`
- 天然支持分支写作（试写不同走向）

**工作量**：1-2 天
**价值**：零成本获得完整版本历史 + 分支能力

---

### ④ MCP Server 模式（开放工具协议）

**来源**：Anthropic MCP 协议成为 2025-2026 年 AI 工具调用标准

**核心思想**：写作助手工坊自身作为一个 MCP Server 暴露工具，任何 MCP Client（Claude Code、Cursor 等）都可以直接调用

**当前状态**：内部有 tools_router.py + tool_definitions.py
**改进方向**：
- 实现 MCP 协议标准端点：`/tools/list`、`/tools/call`
- 支持 `streamable HTTP` 传输
- 让外部 AI Agent 也能调用写作工坊的能力

**工作量**：3-5 天
**价值**：从"内置 AI"扩展到"任何 AI 都可以调用工具"

---

### ⑤ Agentic Workflow（自主写作 Agent）

**来源**：2026 年 Agent 从工具执行进化为自主规划+执行

**核心思想**：不只是"我问你答"，而是 Agent 可以自主规划写作步骤并执行：
1. 分析大纲 → 2. 搜索相关章节 → 3. 起草 → 4. 自检 → 5. 修订 → 6. 完成

**当前状态**：9 阶段工作流（人工驱动）
**改进方向**：在 AI 对话中嵌入 Agentic Loop：
- 用户说"帮我写下一章"
- AI 自动调取大纲 → 读取前章 → 参考写作规范 → 生成 → 自我审查 → 修订

**工作量**：3-5 天
**价值**：从"写作工具"进化为"写作伙伴"

---

### ⑥ WebSocket 实时推送

**来源**：现代 Web 应用标配

**核心思想**：后端状态变化主动推送到前端，而非前端轮询

**当前状态**：完全基于 HTTP 请求-响应
**改进方向**：
- 章节保存后推送到所有打开的面板
- 快照创建后通知前端
- 统计更新后实时刷新

**工作量**：2-3 天
**价值**：消除手动刷新，体验更流畅

---

### ⑦ Tauri 替代 Electron 打包

**来源**：2024-2026 年 Tauri 2.0 成熟

**核心思想**：用系统原生 WebView 替代 Chromium 捆绑，安装包从 100MB+ 降到 5MB

**当前状态**：PyInstaller 打包（45.8 MB）
**改进方向**：
- 前端独立为 SPA
- 用 Tauri 2.0 打包
- 后端 FastAPI 作为 sidecar 进程

**工作量**：1-2 周
**价值**：安装包缩小 90%，启动速度提升

## 三、优先级建议

| 排名 | 方向 | 工作量 | 价值 | 理由 |
|:----:|:-----|:------:|:----:|:------|
| 1 | RAG 2.0 向量知识库 | 2-3 天 | 高 | AI 回答质量提升最直接 |
| 2 | 语义化 Git 集成 | 1-2 天 | 高 | 零成本获得版本历史 |
| 3 | Agentic Workflow | 3-5 天 | 高 | 从工具进化为伙伴 |
| 4 | MCP Server 模式 | 3-5 天 | 中 | 开放生态，长期价值 |
| 5 | WebSocket 实时推送 | 2-3 天 | 中 | 体验提升 |
| 6 | CRDT 同步架构 | 4-6 周 | 低 | 单机场景非刚需 |
| 7 | Tauri 打包 | 1-2 周 | 低 | 当前 exe 够用 |

最推荐的前三个：**向量知识库 → 语义化 Git → Agentic Workflow**，加起来不到两周。要开始吗？
