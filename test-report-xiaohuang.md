## 8. 边界测试综合

### 路径穿越（重要发现）

所有 4 种路径穿越 payload **均返回 201 成功创建项目**：

| 测试 | payload | 结果 |
|------|---------|------|
| Windows 相对路径 | `..\\..\\..\\Windows\\System32` | 201 创建成功 |
| Linux 绝对路径 | `/etc/passwd` | 201 创建成功 |
| Windows 绝对路径 | `C:\\Windows\\System32\\cmd.exe` | 201 创建成功 |
| Null byte 注入 | `../../../etc/passwd%00.txt` | 201 创建成功 |

**风险等级：⚠️ 关注**。虽然项目创建仅存储 path 字符串到 JSON 文件，不直接操作文件系统，但如果后续功能（如扫描目录、读取文件）使用了未清理的 path 参数，可能造成路径穿越漏洞。

## 关键发现汇总

### 🔴 高风险（建议立即修复）

1. **路径穿越未过滤**（第3项，项目管理 - path 字段）
   - `..\\..\\..\\Windows\\System32`、`/etc/passwd`、`C:\\Windows\\System32\\cmd.exe` 均被接受
   - 即使当前只存字符串，建议在入口处对 path 做规范化校验，防止后续扩展后成为漏洞
   - 建议：对 path 做 `os.path.normpath` + 检查是否在允许的基目录内

### 🟡 中风险（建议优化）

2. **导出接口无章节存在性校验**（第6项）
   - `chapter_ids` 传入不存在 ID 后返回 201 且 chapter_count=1
   - 建议：在导出前校验所有 chapter_id 是否属于该项目

3. **XSS 注入无转义**（第8项）
   - name 可包含任意 HTML/JS 代码
   - 建议：后端统一做 HTML 实体转义存储，或前端渲染时自行处理

### 🔵 低风险/建议

4. **章节 batch/delete 错误信息不精确**
   - 不存在的 ID 报"不支持的文件类型"而非"ID不存在"
   - 建议：根据错误类型区分提示

5. **尾斜杠路由不一致**
   - `/projects` 200，`/projects/` 307
   - 建议：统一路由风格

6. **空 list 请求格式**
   - `batch/delete` 和 `batch/export` 拒绝裸数组 `[]`
   - 建议：API 文档中明确说明期望的 body 格式

7. **tools/call 端点不可用**
   - 返回 405，功能尚未就绪
