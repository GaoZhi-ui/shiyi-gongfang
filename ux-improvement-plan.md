## 一、AI写作工具 · 用户抱怨与期望

### 1.1 高频抱怨

| # | 抱怨内容 | 典型用户原话 | 来源 |
|---|----------|-------------|------|
| 1 | 生成内容过于套路化、模板化 | "中后段有点模式化，后面需要自己去打磨" | 搜狐测评 |
| 2 | 人物塑造扁平、情感线粗糙 | "人物性格塑造还不够立体，情感线细节得我自己补" | 搜狐测评 |
| 3 | 像在和客服聊天，没有创作感 | "大部分不是太笼统就是用起来像在和客服聊天" | 什么值得买 |
| 4 | AI写作千篇一律，缺乏辨识度 | "它会按套路推进剧情，少了点新鲜感" | 搜狐/Kimi 实测 |
| 5 | 自由发挥效果差，必须靠提示词死命控制 | "用提示词控制方向，比让AI自由发挥更有效" | 多数用户共识 |
| 6 | 中长篇生成到后半段开始崩 | 多篇评测提到"后半段质量下降" | 知乎/什么值得买 |
| 7 | 隐私焦虑——创意/世界观设定不敢存商业云端 | "不想把自己的创意脑洞存到别人的商业云端" | V2EX Vela 发布帖 |
| 8 | 输出偏"讨论型"而非"文学化" | "它输出的内容偏讨论型，不够文学化" | 通义千问评测 |

### 1.2 用户真实期望

| # | 期望 | 隐含需求 |
|---|------|---------|
| 1 | AI辅助而非替代——帮忙搭框架，细节自己填 | 协作感 > 代劳感 |
| 2 | 能理解和维持已有的故事风格/世界观 | 风格一致性是刚需 |
| 3 | 续写时要"懂"上下文，不要推倒重来 | 上下文记忆深度 |
| 4 | 能辅助角色塑造、伏笔管理式 | 知识管理+写作一体 |
| 5 | 本地优先——数据在自己手里 | 隐私/安全是第一信任门槛 |
| 6 | 定价合理，不要按token算计 | 国内用户对订阅制疲倦 |

### 1.3 从抱怨中提取的模式

> **核心矛盾：AI写作工具在"效率"和"个性"之间走钢丝。用户要的是创作伙伴，不是自动生成器。**
>
> 当前市场表现为：工具越通用，内容越没有辨识度；工具越垂直（如笔灵AI做小说），框架越完整但细节越空洞。
>
> **写作助手工坊的方向机会：以"风格一致性"和"知识管理"为差异化锚点，而不是拼生成速度。**

## 二、写作工具 · 五大类痛点

### 2.1 同步与数据安全

| 痛点 | 用户反馈摘录 | 涉及产品 |
|------|-------------|---------|
| 云同步不可靠，写一半丢稿 | 多名用户提到"闪退后内容丢失""云同步冲突" | 云上写作、橙瓜 |
| 离线状态等于不能写 | 网络差时无法使用，断网断稿 | Notion、在线工具 |
| 对云端存储的信任危机 | "不想把创意脑洞存到别人云端" | 多数SaaS工具 |
| 备份机制不透明 | 用户不知道数据存哪了、怎么备份 | 橙瓜、口袋写作 |

### 2.2 导出与格式兼容

| 痛点 | 用户反馈摘录 | 影响面 |
|------|-------------|--------|
| 导出格式乱码/丢失样式 | Pages导出后格式错乱、内存暴增 | 写作→发布链条断裂 |
| 不支持多格式一键导出 | 每个平台要单独调整排版 | 多平台创作者 |
| 从工具到发布平台转换成本高 | 公众号不支持Markdown，需额外转换 | 所有Markdown工具 |
| 格式锁定（vendor lock-in） | 用了一个工具后数据出不来 | Scrivener、Ulysses |

### 2.3 编辑器体验

| 痛点 | 用户反馈摘录 | 涉及产品 |
|------|-------------|--------|
| 启动慢、反应卡顿 | "WPS启动会先弹模板页，没有其他码字软件快" | WPS |
| 功能太多反而找不到核心功能 | "功能齐全但用不完，还没摸索明白" | WPS、橙瓜 |
| 缺少字数统计/写作目标 | "没有字数统计等功能" | Metion |
| 编辑体验和预览效果不一致 | 分屏/即时渲染差异 | 多数Markdown编辑器 |
| 图片粘贴体验差 | 需要配置图床，粘贴→上传链路长 | Typora+图床方案 |

### 2.4 工具链碎片化

**典型用户配置（来自豆瓣/夜雨聆风调研）：**

> 备忘录 + 百灵创作 + 幕布（大纲）
>
> Obsidian + Typora + flomo + Upic + Pandoc
>
> 作家助手（码字）+ WPS（修文）+ 石墨（协作）

每个用户都用了 3~5 个工具组合，没有一个工具能覆盖"灵感→大纲→写作→修订→发布"全流程。

### 2.5 学习成本与上手门槛

| 痛点 | 用户反馈 |
|------|---------|
| Obsidian全英文界面劝退 | "Obsidian——全英" |
| 插件系统太复杂 | "瑞士军刀级，但也是把双刃剑" |
| 付费门槛 | "没有体验就需要付费"（Ulysses） |
| 功能限制 | "没有会员只能创建一个文档"（WonderPen） |

## 三、开源写作工具 · 用户评价

### 3.1 主流开源/免费写作工具口碑

| 工具 | 用户评价 | 高频优缺点 |
|------|---------|-----------|
| **Obsidian** | 好评居多，但学习曲线陡 | + 本地优先、数据自己管、插件丰富<br>- 全英文界面劝退中文用户、配置复杂、同步需付费 |
| **Typora** | 写作体验最优，但功能单一 | + 所见即所得、极简、丝滑<br>- 需搭配图床/管理工具、无知识管理 |
| **Effie** | 小众但忠诚度高 | + 思维导图适合大纲、极简界面<br>- 码字功能不够、用户少生态弱 |
| **百灵创作** | 分类做得最好的国货 | + 章节/大纲/角色/伏笔分类清晰<br>- 与好好码字同质化 |
| **橙瓜** | 功能全但体验粗糙 | + 拼字功能社交、双备份<br>- iPad不适配、页面不对版 |
| **口袋写作** | 简洁好用的基础款 | + 基础功能完善、页面简洁<br>- 阅读功能要广告解锁/会员 |
| **Writer（LibreOffice）** | 过重，不适合小说写作 | + 免费开源、功能完整<br>- 太重太正式，像在写论文 |

### 3.2 开源写作工具的群体评价模式

```
正面共识：
  本地优先 ✓          数据自主可控 ✓        社区活跃 ✓
  可定制性强 ✓         无订阅制 ✓            长期可用 ✓

负面共识：
  UI/UX粗糙 ✗          中文支持不完善 ✗       上手门槛高 ✗
  缺少写作专项功能 ✗    同步方案需要自建 ✗      生态碎片化 ✗
```

### 3.3 Vela 的启示

V2EX 最近发布的 [Vela](https://www.v2ex.com/t/1207515) ——一款本地优先、隐私安全的 AI 长文/小说创作 IDE——在开发者群体中引起关注。其设计理念验证了：

- **本地优先 + AI 辅助** 是用户的两大刚需
- **隐私焦虑** 是推动用户从 SaaS 回归本地的核心驱动力
- **开源 + 长文创作** 这条赛道正在升温

## 四、写作助手工坊 · 当前现状

参考 `README.md`、`ui-improvements.md`、`security-review.md` 及代码结构：

| 维度 | 现状 |
|------|------|
| 定位 | 《泰拉拾遗录》专属写作环境，AI聊天+写作工具箱+自动审查 |
| 编辑器 | Markdown 编辑器，左侧编辑右侧实时预览 |
| AI 聊天 | 需配置 API Key，对话辅助写作 |
| 审查 | 自动审查：拼写、句式、字数、密度（12项自检） |
| 数据安全 | 完全本地运行（FastAPI + 本机），目前无云同步 |
| 技术栈 | Python FastAPI + HTML/JS 前端 |
| 打包 | PyInstaller 可构建桌面 exe |
| 使用模式 | 对话式 + 编辑面板 |
| 知识整合 | 知识库同步脚本（sync_knowledge.py）从 `knowledge_base/` 复制 .md 文件 |
| 品牌 | 暗色主题，金色(`#e8a84c`)强调色，世界观贴合 |

## 五、综合改进建议

### A. 核心定位巩固（P0）

| 问题（来自调研） | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|----------------|-------------|-------------|---------|--------|
| 写作工具功能零散，需组合 3~5 个工具 | 豆瓣/夜雨聆风 | 已有聊天+编辑+审查+知识库，四合一 | 强化"全流程覆盖"定位——用户在一个界面完成从灵感到发布。明确告知用户"你不需要其他工具"。 | P0 |
| AI写作"像客服聊天"缺少创作感 | 什么值得买/搜狐 | 聊天界面为基础对话式 | 将AI角色从"问答助手"调整为"写作搭档"——引入"续写""润色""节奏诊断""伏笔检查"等写作专项能力，对话模板预设 | P0 |
| 隐私焦虑让用户不敢用云端工具 | V2EX | 完全本地运行 | 将"本地优先"作为核心卖点在 README 和界面中凸显。可考虑加入"无网络也可使用"状态标识 | P0 |

### B. 写作体验优化（P0~P1）

| 问题 | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|------|-------------|-------------|---------|--------|
| AI写作后半段质量下降、模式化 | 搜狐/知乎 | AI聊天未限定写作范围 | 引入"写作上下文管理"——每次对话带上当前章节全文、写作风格指南、人物档案，让AI"知道你在写什么" | P0 |
| 风格一致性差，AI不懂已有设定 | 什么值得买/V2EX | 已有知识库同步 | 增强知识库关联：人物档案、伏笔状态、世界观设定自动注入 AI prompt，每次回答前自动加载上下文 | P0 |
| 缺少字数统计和写作目标 | 夜雨聆风/豆瓣 | 状态栏有字数显示 | 增加章节目标进度条（2500±300 基线），增加写作时长/专注计时器，底栏实时反馈 | P1 |
| 导出格式受限 | 多数用户 | 有聊天记录导出 | 增加多格式导出：Markdown → docx（写作→发布），增加 Pandoc 集成或内置转换 | P1 |
| 无法离线使用 | 同步/网络类痛点 | 纯本地无云依赖 | 已经是本地，无需改动。但在启动界面增加"离线模式"标识，给用户安心感 | P1 |

### C. 工具链整合（P1~P2）

| 问题 | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|------|-------------|-------------|---------|--------|
| 灵感碎片无处记录 | 少数派/flomo 用户 | 仅有聊天输入 | 增加"灵感便签"——浮动窗口/B侧栏，随时记录碎片，后期可拖入正文或转为章节笔记 | P1 |
| 大纲/结构管理缺失 | 幕布/Effie 用户 | 无大纲视图 | 增加右侧"大纲"面板（基于 Markdown 标题自动提取），支持拖拽调整章节顺序 | P1 |
| 多章节管理不便 | 豆瓣/百灵创作 | 通过文件系统管理 | 改进章节导航：章节列表（左栏）+ 当前章节编辑（中栏）+ 关联知识（右栏），支持快速跳转 | P1 |
| 没有版本历史 | 通用痛点 | 无 | 集成 Git 或本地快照机制：每次保存自动生成快照，支持回溯/对比 | P2 |

### D. AI 能力专项提升（P1~P2）

| 问题 | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|------|-------------|-------------|---------|--------|
| 角色塑造需要人工补 | 搜狐/多个AI评测 | 仅有聊天 | 增加"角色诊断"专项：粘贴角色段落 → AI 分析角色立体度/一致性/弧光 | P1 |
| 伏笔管理缺失 | V2EX/写作社区 | 无 | 增加"伏笔追踪"面板：登记→跟踪→回收标记，AI自动检测遗漏伏笔 | P1 |
| 对"写不出来"没有帮助 | 多个用户提到卡文 | 仅有续写功能 | 增加"卡文急救"模式：给出 3 种不同走向的低门槛选择，降低重启写作的心理阻力 | P2 |
| 写作风格一致性维护 | 用户公认痛点 | 风格指南为静态知识 | 将写作风格指南（v3准则）注入每条 AI 回复的 system prompt。每次输出自动校验：是否匹配 2500±300 字数、是否避免了说教、是否留白 | P1 |

### E. 数据安全与信任（P1）

| 问题 | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|------|-------------|-------------|---------|--------|
| 不知道数据存在哪 | V2EX 隐私焦虑 | 本地文件，用户可见 | 在界面增加"数据位置"指示器：显示当前存储路径、文件数量、最后备份时间 | P1 |
| 没有一键备份 | 通用痛点 | 手动复制 | 增加"一键备份"——打包所有 .md 文件 + 知识库 + 聊天记录为 zip，可选择备份路径 | P1 |
| API Key 管理不透明 | V2EX 安全讨论 | settings 配置 | 增加 API Key 安全提示：明确告知 key 仅存本地、不上传、使用用途 | P1 |

### F. 品牌与差异化（P2~P3）

| 问题 | 用户反馈来源 | 写作助手工坊现状 | 改进建议 | 优先级 |
|------|-------------|-------------|---------|--------|
| 大部分写作工具没有温度 | 少数派 | 有品牌感但不够 | 强化"写作助手工坊"世界观人设：加载文案改为"翻阅尘封的档案""在泰拉地图上标记"，错误页显示世界观插图 | P2 |
| 缺少社区/分享入口 | 网文作者社交需求 | 纯单机 | 增加"导出并分享"流程：一键导出排版好的 docx，附带发布到知乎/公众号的适配建议 | P3 |

## 六、优先级路线图

| 阶段 | 时间估计 | 改进项 | 目标 |
|------|---------|--------|------|
| **Phase 1** | 1~2天 | A(核心定位) + B(写作体验) + 知识库强关联 | 让AI真正"懂"你在写什么 |
| **Phase 2** | 2~3天 | C(工具链整合) + D(AI专项) | 覆盖全写作流程 |
| **Phase 3** | 1~2天 | E(数据安全) + F(品牌差异化) | 建立信任 + 独特气质 |

### Phase 1 具体任务清单

1. **AI 上下文增强**：每次对话自动注入当前章节全文 + 风格指南 + 人物档案（`knowledge/` 目录自动加载）
2. **写作专项命令**：预设"续写"、"润色"、"节奏诊断"、"伏笔检查"四个快捷按钮，替代通用聊天
3. **字数目标进度条**：状态栏增加 `当前字数/目标字数` 进度条，实时显示
4. **章节导航改进**：左栏增加按章节组织的文件列表，支持快速切换
5. **本地优先标识**：启动界面/底栏增加"离线可用"标识
