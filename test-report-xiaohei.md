## 7. 暗色主题

| 项目 | 测试步骤 | 实际结果 | 通过 |
|------|---------|---------|:----:|
| 整体配色 | 检查 CSS 颜色变量 | 背景层次分明：body `#141416` → sidebar `#1a1a1a` → inputs `#1e1e1e`，暖暗色调 | ✅ |
| 金色是否刺眼 | 检查金色使用场景 | 金色 `#d4a040` 为暖金（非亮金/荧光），仅用于 logo/active/focus 状态，辅以 `rgba(212,160,64,0.1~0.25)` 半透明背景 | ✅ |
| 文本可读性 | 检查字体颜色对比度 | 主文本 `#c8c3b8`（暖灰）、标题 `#d4d4d4`（稍亮）、次要 `#888/999/666`，层次分明 | ✅ |
| 边框/分割线 | 检查分隔线颜色 | `#2a2825` — 暖暗色细线，不跳脱 | ✅ |
| 选择高亮 | 检查 selection 样式 | `rgba(212,160,64,0.25)` 暖金色半透明，不刺眼 | ✅ |
| 滚动条 | 检查滚动条样式 | 4px 宽，thumb `#333` → hover `#444`，低调不干扰 | ✅ |

**配色方案总结：**

```
背景层:  #141416 (body)
         #1a1a1a (sidebar / panel)
         #151515 (topbar / editor-footer)
         #1e1e1e (inputs / selects / textareas)
金色:    #d4a040 (logo / active / focus / save-btn)
         rgba(212,160,64,0.1) (active bg)
         rgba(212,160,64,0.25) (selection)
文本:    #d4d4d4 (headers / hover)
         #c8c3b8 (body 主文本)
         #999 / #888 / #666 (次要文本 / 提示)
边框:    #2a2825 (分割线 / 边框)
         #242424 (hover 背景)
状态:    #5cb85c (绿色-已保存)
         #f0ad4e (琥珀-保存中)
         #d9534f (红色-错误)
```

> 暖金色调与整体暗色主题搭配协调，不刺眼且足够醒目。

### 部分通过项

| 功能 | 状态 | 原因 |
|:----|:----:|:-----|
| Ctrl+N 新建章节 | ⚠️ | 快捷键绑定存在但 `prompt()` 在 headless 中自动取消 |

### 发现的问题

1. **编辑器按钮点击异常**：`#editorToggleBtn` 在 Playwright 中坐标为负值（`top:-394, left:-401`），疑似页面滚动同步问题。通过 JS 直接操作 DOM 属性可绕过。
2. **部分功能依赖后端 API**：角色 / 标签筛选 / 全局搜索等功能在无后端数据时无法展示实际内容，建议配置 API Key 后复测。
3. **Ctrl+N 在 headless 限制**：`prompt()` 弹窗在 headless 中不支持，建议后续改为自定义模态框实现新建章节对话框。
4. **「全卷细纲」「时间线表」「伏笔追踪」未测试**：超出本次测试范围，留待后续。
