# 写作工具前端设计与用户体验模式研究报告

> 生成日期：2026-05-21
> 研究范围：Typora / iA Writer / Scrivener / Novelcrafter / 蛙蛙写作 / 明文写作 / 墨斗写作
> 格式：功能 | 工具来源 | 实现思路 | 引入优先级 | 预估实现难度

## 一、编辑体验层

| 功能 | 工具来源 | 实现思路 | 引入优先级 | 预估实现难度 |
|------|---------|---------|-----------|-------------|
| 所见即所得 Markdown 实时渲染 | Typora | 基于 Electron webview，监听内容变化（`input`/`MutationObserver`），实时解析 Markdown AST 并渲染为带样式的 HTML。编辑区用 `contentEditable`，光标位置通过选区映射回源码位置。非开源，底层可能使用自定义 Markdown 解析器（类似 marked/markdown-it 的 AST → DOM 管线） | P0 | ⭐⭐⭐ 较高 |
| 专注模式（Focus Mode） | iA Writer | 高亮当前句子/段落，其余内容半透明淡化。实现：遍历文本节点，判断当前选区所在行/段，相邻节点全局降低 `opacity`。iA Writer 进一步关联打字机滚动，光标始终保持在屏幕中央 | P0 | ⭐ 低 |
| 打字机滚动（Typewriter Scrolling） | iA Writer | 编辑时自动滚动视图，使当前行保持在屏幕垂直中央。实现：监听光标位置变化，计算相对视口的偏移量，调用 `scrollIntoView({ block: 'center' })` 或手动调整 `scrollTop` | P1 | ⭐ 低 |
| 语法高亮（Syntax Highlight） | iA Writer | 借鉴代码编辑器思路，对文本进行词性标注（名词/动词/形容词/副词），用不同颜色区分。实现：在前端调用自然语言分词库（如 compromise/compendium），标注词性后通过 CSS class 染色 | P2 | ⭐⭐⭐ 较高 |
| 风格检查（Style Check） | iA Writer | 标记陈词滥调、填充词、冗余修饰语。实现：基于规则引擎（正则+词库匹配），本地运行无网络请求。被标记内容添加删除线下划线，点击后可移除 | P2 | ⭐⭐ 中等 |
| 无干扰全屏模式 | iA Writer / Typora | 隐藏菜单栏、状态栏、滚动条等所有 UI 组件，只保留纯编辑区。实现：壳层切换为无边框窗口，CSS 隐藏所有装饰性元素 | P0 | ⭐ 低 |
| 源/预览分屏 | iA Writer / Typora | 左侧 Markdown 源码，右侧渲染预览，同步滚动。实现：双面板布局，左侧监听滚动同步右侧滚动百分比，右侧监听同样策略反向同步 | P3 | ⭐ 低 |
| 粘贴智能过滤 | Typora | 粘贴时自动识别来源格式（HTML / 纯文本 / 富文本），转换为目标格式。实现：剪贴板 API 读取多种 MIME type，按优先级选择并调用对应 parser | P2 | ⭐⭐ 中等 |
| 快照版本管理 | Scrivener | 单个文档的版本存档，支持对比差异。实现：存储每次快照的内容副本，差异对比用 diff 算法（如 Myers diff），分屏显示新旧版本 | P3 | ⭐⭐⭐ 较高 |
| 文本快照对比 | Scrivener | 在校对助手中并排显示两版差异。实现：双列 `diff` 视图，增删内容高亮着色 | P3 | ⭐⭐ 中等 |

## 二、项目组织结构

| 功能 | 工具来源 | 实现思路 | 引入优先级 | 预估实现难度 |
|------|---------|---------|-----------|-------------|
| 活页夹（Binder） | Scrivener | 左侧树形层级结构，支持文件夹/子文件夹/文档的嵌套，拖拽排序。实现：递归树组件 + drag & drop API，每个节点存储 UUID、parentId、sortOrder | P0 | ⭐⭐ 中等 |
| 索引卡/公告板（Corkboard） | Scrivener | 可视化卡片面板，每张卡片代表一个场景/章节，展示标题和提要。支持自由模式（任意排列）和标签模式（按颜色分类）。实现：虚拟网格布局 + 拖拽定位，卡片数据同步 binder 节点 | P0 | ⭐⭐⭐ 较高 |
| 大纲视图（Outline） | Scrivener | 表格化展示文档结构，列可自定义（标题、提要、字数、状态、修改日期等）。实现：可编辑表格组件，单元格直接修改元数据，支持批量操作 | P1 | ⭐⭐ 中等 |
| 组合式文本（Scrivenings） | Scrivener | 将文件夹内的所有子文档拼接为连续文本，统一审阅。实现：递归遍历子节点，拼接文本内容并插入灰色虚线分隔符，修改时 sync 回原文档 | P1 | ⭐⭐ 中等 |
| 非线性写作 | Scrivener | 允许按任意顺序写场景，后续通过拖拽重组。实现：每个场景是独立文档节点，不依赖顺序——渲染和导出时按排序字段组合 | P0 | ⭐⭐ 中等 |
| 三视图布局（编辑+大纲+资料） | Scrivener / Novelcrafter | 同时打开编辑器、大纲/卡片视图、资料参考面板。实现：可拖拽分栏布局（如 Golden Layout 或 React SplitPane），每个面板可独立切换内容 | P1 | ⭐⭐⭐ 较高 |
| 梗/脑洞库 | Scrivener | 独立的文件夹结构存放未完善的灵感，支持链接外部文件（图片、视频、PDF）。实现：虚拟文件系统 + 资源链接管理器 | P2 | ⭐⭐ 中等 |

## 三、资料与世界观管理

| 功能 | 工具来源 | 实现思路 | 引入优先级 | 预估实现难度 |
|------|---------|---------|-----------|-------------|
| Codex 世界档案库 | Novelcrafter | 结构化数据库存储人物、地点、物品、事件，支持 AI 写作时引用。实现：关系型/文档型 Schema，每条记录含名称、描述、关联关系、标签；写作侧通过 `@人物名` 或弹出选择器引用 | P0 | ⭐⭐⭐⭐ 高 |
| 人物/地点/事件卡片 | Novelcrafter / Scrivener | Codex 中每条记录的可视化卡片，显示摘要信息，支持关联跳转。实现：卡片组件 + GraphQL 查询 | P1 | ⭐⭐ 中等 |
| 人物出场时间线 | Novelcrafter | 展示各角色在各章节的出现分布。实现：从文本中提取人物提及，生成时间线/甘特图可视化 | P2 | ⭐⭐⭐ 较高 |
| 关联图/关系图谱 | Novelcrafter | 人物-地点-事件的关系网络可视化。实现：力导向图（D3-force / vis-network），节点可点击跳转详情 | P3 | ⭐⭐⭐⭐ 高 |
| 快速参考面板 | Scrivener | 右侧栏分"提要"和"便笺"，可在编辑时随时参考。实现：右侧滑出面板，内容与当前文档关联存储 | P1 | ⭐ 低 |
| 写作进度/字数统计 | Scrivener | 每文档、每文件夹、整个项目的字数统计和目标管理。实现：递归统计算法 + 目标设定 UI（图形进度条） | P0 | ⭐ 低 |

## 四、AI 协作层

| 功能 | 工具来源 | 实现思路 | 引入优先级 | 预估实现难度 |
|------|---------|---------|-----------|-------------|
| AI 续写/补全 | Novelcrafter / 蛙蛙写作 | 基于当前上下文生成后续内容。实现：取光标前 N 个 token 作为 prompt（含 Codex 上下文引用），调用 LLM API 流式返回 SSE，逐步渲染到编辑器 | P0 | ⭐⭐⭐ 较高 |
| 场景节拍（Beats）引导 | Novelcrafter | 预设场景的情绪/节奏变化点，AI 按 beats 生成更符合叙事弧线的内容。实现：结构化标签（紧张/舒缓/冲突/揭示），作为 prompt 模板的元数据输入 | P1 | ⭐⭐⭐ 较高 |
| AI 聊天/角色对话模拟 | Novelcrafter | 内置聊天面板，与 AI 讨论角色动机、情节逻辑。实现：独立的 chat UI，对话历史作为上下文，可提取对话中的创意存入 Codex | P1 | ⭐⭐ 中等 |
| Codex 感知的 AI 生成 | Novelcrafter | AI 在写作时自动调用 Codex 中的设定信息，保持一致性。实现：RAG 模式——生成前检索相关 Codex 条目作为 prompt 的 system 上下文 | P0 | ⭐⭐⭐⭐ 高 |
| AI 写作归属标记 | iA Writer | 对 AI 生成的文本用颜色标记区分，显示来源透明度。实现：在 text model 中为文本片段附加 `origin` 元数据（human/AI/claude/chatgpt），渲染时应用对应 CSS class | P2 | ⭐⭐ 中等 |
| 全链路 Agent 工作流 | 蛙蛙写作 | 小说→剧本→视频，一站式转换。实现：多个 Agent 管道串联，每个阶段有独立的 prompt 模板和转换规则 | P2 | ⭐⭐⭐⭐ 高 |
| 多 AI 模型一键切换 | 蛙蛙写作 | 同一界面调用不同底层模型。实现：抽象 LLM 适配层，统一 I/O 接口，按模型 API 分发请求 | P1 | ⭐⭐ 中等 |
| 网文专用提示词库 | 蛙蛙写作 | 内置网文类型专属 prompt 模板。实现：按类型（玄幻/言情/悬疑）组织提示词库，UI 侧提供一键应用按钮 | P1 | ⭐⭐ 中等 |
| AI 漫剧（文字→漫画视频） | 蛙蛙写作 | 将小说内容自动转换漫画视频。实现：分镜拆分 → 角色一致性绘制（Stable Diffusion） → TTS 配音 → 视频合成 | P3 | ⭐⭐⭐⭐⭐ 极高 |

## 七、中文写作工具专项分析

### 蛙蛙写作（杭州引力智航科技 / 波形智能）

| 功能 | 实现思路 | 亮点 | 参考价值 |
|------|---------|------|---------|
| 全链路 Agent 工作流 | 多阶段 pipeline：构思→大纲→正文→剧本→视频 | 一站式完成，减少工具切换成本 | AI 写作工具可参考其 Agent 编排模式 |
| 多 AI 模型聚合 | 统一调用层，单账户切换不同模型 | 覆盖 200+ 写作场景 | 可作为多模型架构的参考 |
| 网文场景适配 | 基于网文数据训练，内置风格模板 | 中文网文生成质量显著高于通用模型 | prompt 模板库管理方式值得借鉴 |
| AI 漫剧 | 文字→分镜→角色→配音→视频 | 全自动转化 | 多模态扩展能力，但实现成本极高 |
| 多端同步 | Web/PC/iOS/Android/macOS 云同步 | 无缝切换 | 基础能力，写作工具的标配 |

### 明文写作 & 墨斗写作

> **明文写作**——未在主流渠道找到独立产品信息。概念上可能与"明文"相关（纯文本优先、格式透明），或是某个小众工具/用户社区中的创作方法。建议确认具体指代的产品或开源项目。
>
> **墨斗写作**——目前搜索引擎结果全部被木工工具和格斗手游覆盖。若指代某一 AI 写作工具，可能为较新或已更名的产品。建议从中文写作社区（晋江论坛、龙空、知乎写作版块）进一步求证。

若这两个工具是用户社区中流传的特定概念，以下是从名称推断的功能方向：
- **明文写作**：可能强调"明"（透明）和"文"——写作过程可视、AI 介入可追溯、格式无黑箱。与 iA Writer 的 Authorship 理念相似。
- **墨斗写作**："墨斗"是木工取直的墨线工具——可能寓意写作中的"取直"（去 AI 味、修正偏差）。与"去AI味"的需求方向一致。

## 八、综合优先级排序（建议实施路线）

### 第一阶段（P0 — 核心体验，必须实现）
| 排名 | 功能 | 理由 |
|------|------|------|
| 1 | 专注模式（Focus Mode） | 实现成本低，用户感知强，写作工具的核心卖点 |
| 2 | 打字机滚动 | 与 Focus Mode 搭配，沉浸体验的基础 |
| 3 | 无干扰全屏 | 几乎零成本，立竿见影的体验提升 |
| 4 | 活页夹（Binder 树形结构） | 项目组织的基础设施 |
| 5 | 索引卡/公告板（Corkboard） | 可视化大纲，长篇写作的核心需求 |
| 6 | 所见即所得 Markdown 实时渲染 | 对标 Typora 的用户预期 |
| 7 | 写作进度/字数统计 | 基础但重要的反馈 |
| 8 | Codex 世界档案库 | AI 写作一致性的核心支柱 |
| 9 | AI 续写/补全 | 最基础的 AI 协作能力 |
| 10 | Codex 感知的 AI 生成 | 确保 AI 输出不崩设定的关键 |
| 11 | 纯 CSS 主题系统 | 可定制性的基础，扩展性高 |
| 12 | 暗色模式 | 基本要求 |

### 第二阶段（P1 — 重要但不紧急）
- 非线性写作
- Codex 可视化卡片
- 快速参考面板
- 大纲视图
- 组合式文本审阅
- 多 AI 模型切换
- 网文提示词库
- 场景节拍引导
- AI 聊天/角色模拟
- 自定义字体与排版
- Markdown → PDF/DOCX 导出
- 三视图布局
- 自定义暗色模式

### 第三阶段（P2 — 加分项）
- 语法高亮（词性标注）
- 风格检查
- 粘贴智能过滤
- 人物出场时间线
- 梗/脑洞库
- AI 写作归属标记
- 全链路 Agent 工作流
- 自定义模板导出

### 第四阶段（P3 — 远期规划）
- 源/预览分屏
- 快照版本管理
- 文本快照对比
- 关联关系图谱
- AI 漫剧
- 直接发布到博客/平台
- 主题市场

## 九、实现难度说明

| 难度级别 | 预估工作量 | 典型场景 |
|---------|-----------|---------|
| ⭐ 低 | 1-2 天 | 纯 CSS 实现、基础 DOM 操作、单组件功能 |
| ⭐⭐ 中等 | 3-7 天 | 涉及数据模型设计、多组件协作、简单算法 |
| ⭐⭐⭐ 较高 | 1-3 周 | 涉及渲染管线、LLM 集成、复杂交互 |
| ⭐⭐⭐⭐ 高 | 3-6 周 | 涉及数据库设计、多系统协作、性能调优 |
| ⭐⭐⭐⭐⭐ 极高 | 6 周+ | 涉及多模态模型、视频管线、大规模系统 |
