# 拾遗工坊 · 界面改进建议

> 调研日期：2026-05-20
> 范围：暗色主题AI对话工具 + 中文写作工具UI模式 + 对话式信息层级优化
> 参考：Claude.ai, ChatGPT, DeepSeek, Gemini, Notion AI 等主流产品界面设计

---

## 一、调研总结

### 1.1 2025-2026 暗色主题AI工具界面趋势

**配色体系：**
- 背景从纯黑( `#000` )转向深灰( `#1a1a1a` ~ `#1e1e1e` )，减轻视觉疲劳
- 主文本不再追求纯白，暖灰( `#e0e0e0` ~ `#d0d0d0` )成为标配
- 强调色从一种主色下沉为"主色 + 辅助色"双层体系，降低主色使用面积
- 毛玻璃效果(backdrop-filter: blur)用于模态、浮动面板、提示条

**布局趋势：**
- 三栏/两栏自适应：窄屏自动折叠侧栏
- 右面板从"可有可无"升级为"上下文面板"，功能权重提升
- 聊天区采用**居中窄栏**(max-width: 680~760px)，而非全宽拉伸
- 工具栏采用**图标+文字**紧凑排布，减少垂直空间占用

**动效与反馈：**
- 微交互成为标配：hover缩放(1.01~1.02)、消息入场缓动、加载骨架屏
- 打字指示器从简单圆点升级为智能进度提示
- 滚动锚定( scroll anchoring )防止流式输出时跳转

### 1.2 中文写作工具常见UI模式

**编辑器类(如 Typora, Obsidian)：**
- 左导航 + 中编辑 + 右预览/大纲 三栏
- 所见即所得实时渲染
- 暗色主题下金色/暖橙作为书写光标和选中高亮色

**AI写作辅助类(如 Notion AI, 阅文写作)：**
- 对话式写作界面正在取代传统编辑器
- 侧边栏承载：写作目标设定、人物库、设定集、词汇表
- 内联建议与气泡建议并存
- 写作统计(字数/进度/时间)常驻底栏

### 1.3 对话式工具信息层级优化

**核心原则：**
- 每一屏用户的焦点不超过3个区域
- 操作按钮相对于内容以**6:4的透明度比例**衰减（操作弱化、内容突出）
- 同一层级的信息使用相同的视觉重量（字号/字重/颜色一致）

**主流做法：**
- 用户消息与AI消息使用不同的背景色和边框处理
- AI消息中穿插可操作组件（展开/折叠、复制、重新生成）
- 长消息默认折叠，保留3~4行预览 + 展开按钮
- 消息时间戳淡化到极小字号(10px)，不干扰阅读流

## 二、当前界面诊断

基于 `static/index.html` 的分析：

| 维度 | 现状 | 评价 |
|------|------|------|
| 配色 | `#111` 底色 + `#e8a84c` 金色强调 | 基础扎实，但金色使用面积过大 |
| 布局 | 280px左栏 + 弹性中栏 + 360px右栏 | 三栏结构合理，右栏权重偏低 |
| 字体 | 14px正文，行高1.7 | 中文正文偏小，建议15~16px |
| 动效 | 仅消息入场 + 悬停变色 | 微交互不足，缺少反馈层次 |
| 间距 | 内边距合理，但有优化空间 | 消息气泡和区域边界可更通透 |
| 右面板 | 内容为静态占位符 | 远未发挥应有作用 |
| 输入区 | 底部固定，支持Shift+Enter | 可用但缺少辅助功能 |

## 三、改进建议

### 3.1 配色体系 —— 降低金色饱和度，建立层级灰度

**问题：** 当前金色(`#e8a84c`)在主品牌、活跃态、强调、悬停等多处复用，视觉疲劳。

**建议：**

```
基础色阶调整：
  #111         →  #131417    背景底板（微暖深灰，减轻纯黑的冷感）
  #1a1a1a      →  #1a1d23    面板底色（原代码已接近，保持）
  #151515      →  #181a20    顶栏/底栏/输入区（微调）
  #1e1e1e      →  #22252d    卡片/气泡/输入框（区分度更清晰）
  #2a2a2a      →  #2e3038    边框/分割线（比原亮一些，减少压抑感）

强调色分层：
  主色  #e8a84c  →  保留，降低使用频次，仅用于：
    · 品牌 logo
    · 激活态图标 / 选中态
    · 主要操作按钮（发送）
    · 高优先级标记
  辅色  #d4a043  →  悬停态 / 二级强调
  背景色 rgba(232,168,76,0.08) → 用户消息气泡背景（微金透底）
  
文字色阶：
  主文  #ccc   →  #d4d4d4  略微提亮
  辅文  #888   →  #9a9a9a  可读性提升
  禁用  #555   →  #5a5a5a  保持区别
  标签  #666   →  #707070  次要信息
```

### 3.2 间距与呼吸感 —— 让内容透口气

**问题：** 当前内边距偏紧，在宽屏下内容显得"挤在中间"。

**建议：**
```
聊天区：
  · padding: 16px 20px → 20px 24px (横向)
  · 窄屏(<768px) → 回退到 14px 16px
  
消息气泡间距：
  · margin-bottom: 20px → 24px（增加消息之间的呼吸感）
  · 同一组连续消息（人→人 / AI→AI）间距缩减到 12px
  
用户消息：
  · max-width: 80% → 维持，但气泡 padding: 10px 14px → 12px 18px
  
侧栏项：
  · padding: 7px 12px → 8px 14px
  · 分组标题上边距 10px → 14px
```

### 3.3 微交互与动效 —— 让界面有"手感"

**问题：** 目前仅消息入场有动画，操作反馈局限于变色。

**建议：**
```
悬停动效：
  · tool-item:hover → transform: translateX(2px)（左栏项向右微移）
  · 发送按钮 hover → 加一个微弱脉冲光晕（box-shadow glow）
  · 侧栏折叠按钮 hover → 旋转动画

消息入场：
  · 保持现有 fadeInUp（0.2s ease-out），但增加 stagger（多条消息时逐条延迟 50ms）
  · AI 流式输出时，每个段落 fadeIn 而不是整块出现

面板切换：
  · 右面板内容切换 → 内容做 0.15s fade 过渡，而不是硬切
  · 模态打开 → backdrop 淡入 0.2s + 面板 scaleIn(0.98→1) 0.25s

状态反馈：
  · 发送按钮 loading → 内部图标旋转，而不是整体 disabled+灰显
  · 红色/绿色状态圆点 → 增加呼吸动画（pulse），吸引注意力
```

### 3.4 信息层级重构 —— 让用户一眼找到重点

**问题：** 当前所有消息视觉重量相近，用户/AI区分主要靠颜色和定位。

**建议：**
```
消息层级模型（权重从高到低）：
  1. AI 操作结果 / 审查报告 / 数据 —— 最高权重，加顶部标记条
  2. AI 正文回复 —— 正常权重
  3. 用户输入 —— 弱权重，甚至可以考虑用户消息字体下调到 13px
  4. 系统提示 / 辅助信息 —— 最弱权重，11px 灰色

视觉区分：
  · AI 消息左侧增加 3px 金色竖条（border-left），增强辨识
  · 用户消息右对齐 + 浅金底，AI 消息左对齐 + 深灰底
  · 审查结果中的 pass/fail/warn 使用对应色块标签，而非文本
  
侧栏分组优化：
  · 当前 5 个分组 → 压缩为 3 组：写作流程 / 知识库 / 章节管理
  · "快捷操作"下的"开始写作"和"导出草稿"合并到底栏浮动按钮区域
```

### 3.5 右面板升级 —— 从"展示器"到"上下文引擎"

**问题：** 右面板目前只是被动展示区，加载占位符文案，没有实际交互。

**建议：**
```
通用改进：
  · 面板切换时保留历史状态（切换工具→再切回，保持上次滚动位置）
  · 面板宽度可拖拽调整（从 280px 到 480px）
  · 面板标题区域增加 pin/unpin 按钮，固定为常驻视图

各面板具体升级：

  自检清单面板：
    · 每条增加"重新检查"按钮，无需回聊天区输入
    · 增加分类统计：进度条 + 通过率环图

  写作风格面板：
    · 改为可编辑模式：点击准则可展开详情
    · 增加"应用此准则到当前章节"快捷按钮

  人物档案面板：
    · 搜索/筛选功能
    · 点击人物名→在聊天区预填充查询

  伏笔追踪面板：
    · 带状态进度的时间线视图
    · 每个伏笔可标记"已回收/进行中/未使用"
```

### 3.6 输入区增强 —— 不止是打字

**问题：** 输入区功能单一，只有打字和发送。

**建议：**
```
功能扩展：
  · 输入框左侧增加 @ 快速引用（点击 → 弹出最近对话选择器）
  · 输入框右键菜单或上方工具栏：插入模板 / 粘贴格式化内容
  · 输入框底部增加字符/字数实时统计（带章节目标进度）

对快速操作的支持：
  · 保留"运行审查""粘贴"等快捷文本按钮，但改为 图标+文字 按钮样式
  · 增加"继续写作"按钮（基于上次对话续写）
  · 增加"重试"按钮（重新生成最后一次回复）

多行体验优化：
  · 输入框默认 2 行高，随内容增加到 6 行，超过则滚动
  · Enter 发送 / Shift+Enter 换行 依然是标准，但增加一个 toggle 提示
```

### 3.7 顶栏精简 —— 降低认知负荷

**问题：** 顶栏信息偏多（模型选择、章节徽章、API按钮、清空、新建），视觉杂乱。

**建议：**
```
精简方案：
  · 模型选择器改为更紧凑的下拉（精简为只显示模型名，不显示 type="select" 的完整宽度）
  · 章节徽章移动到左栏底部或底栏
  · API 设置按钮改为齿轮图标，放入顶栏最右侧

可选：
  · 顶栏高度从 50px 降到 44px（目前设计偏桌面包袱）
  · 清空按钮 + 新建按钮 合并到"会话"下拉菜单中
```

### 3.8 状态栏 —— 从"占位"到"信息中枢"

**问题：** 底栏目前只显示连接状态 + 模型名 + 消息/字数，信息利用率低。

**建议：**
```
左半区（连接 + 状态）：
  · 保持 dot 指示灯，但增加悬停 tooltip 显示详细信息
  · 增加写作时间/专注计时器（类似番茄钟）

右半区（统计信息）：
  · 当前章节字数 / 目标字数（2500±300） → 进度条微缩显示
  · 当前会话消息数 → 改为"对话深度：N 轮"
  · 增加一个小型"写作建议"状态：如"本次写作已持续 45 分钟"
```

### 3.9 响应式与自适应 —— 让窄屏体验不掉队

**问题：** 现有响应式只做了 1200px 和 900px 两个断点，且 900px 以下右面板靠绝对定位覆盖。

**建议：**
```
断点策略（更细腻）：
  ≥1400px   → 完整三栏，右面板 380px
  1200~1400 → 右面板 320px，内容区居中 max-width: 760px
  900~1200  → 右面板默认折叠（可展开覆盖）
  600~900   → 左栏改为图标工具栏（宽度 56px，只有图标）
  <600px    → 全屏单栏，左栏和右栏均为抽屉式
  
其他：
  · 左栏可配置为"自动隐藏"模式（类似 VS Code 的侧栏自动隐藏）
  · 聊天区在窄屏时移除左右 padding，全宽利用
  · 状态栏在窄屏时折叠右侧统计，只保留连接状态
```

### 3.10 品牌感与人文细节

**建议增加以下细节，强化"拾遗工坊"的品牌气质：**

- 空白态页面：当没有对话时，展示一句随机"泰拉拾遗录"世界观短句 + 一个微动效图标（如飘浮的沙粒或卷轴）
- 加载文案：替换"正在思考…"为符合世界观的状态提示，如"翻阅尘封的档案…""在泰拉的地图上标记…"
- 时间戳格式：改为"刚才""3分钟前""2小时前"的相对时间格式，更有人情味
- 滚动条：自定义滚动条轨道色为深棕（`#2a251e`），thumb 为金色（`rgba(232,168,76,0.3)`），贴合主题
- 错误态：API 连接失败时，显示一只"迷路的源石虫"插画 + 文案"通讯线路受阻，请稍后再试"
